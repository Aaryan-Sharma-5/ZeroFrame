## Deployed topology

```
User uploads video
       │
       ▼
Vercel (Next.js)          — zero-latency frontend, no video proxying
       │
       ▼ POST /upload
Render (FastAPI)          — uploads video to 0G Storage via Go CLI
       │                    returns merkle root_hash + storage_cid
       ▼ POST /process
Render (FastAPI)          — writes "pending" to Redis, fires Modal HTTP trigger
       │
       ▼
Modal (T4 GPU)            — serverless, spins up on demand
  ├── detect_events()     — audio spike detection + YOLO object detection
  ├── splice_clips()      — ffmpeg cuts highlight clips
  ├── upload_clips()      — pushes clips to 0G Storage (Go CLI)
  └── generate_caption()  — 0G Compute Router (minimax-m3) captions each clip
       │
       ▼ writes to Redis
Render Redis              — job state: pending → processing → complete
       │
       ▼ GET /status/{job_id} (polls every 3s)
Browser                   — shows clips + captions when complete
```

## Demo flow (step by step)

### 1. Open the health endpoint first (~30s)
```
https://zeroframe-backend.onrender.com/health
```
Show the judges:
```json
{
  "status": "ok",
  "redis": { "redis_ok": true, "modal_configured": true },
  "zg_storage_node": "https://indexer-storage-testnet-turbo.0g.ai"
}
```
**Say:** *"Our FastAPI backend is live on Render, Redis is connected, and the Modal GPU worker is registered."*

### 2. Upload a video (~1 min)
- Go to the frontend.
- Upload a short sports clip — 30–60s is ideal for demo speed.
- **Say:** *"The video is being uploaded directly to 0G Storage — a decentralized storage network. We get back a merkle root hash that permanently identifies this content on-chain."*
- Show the returned `root_hash` (starts with `0x...`).

### 3. Watch the job process (~2–4 min)
The frontend polls `/status/{job_id}` every 3 seconds.

| Status | What to say |
|---|---|
| `pending` | *"Job is queued, Modal is cold-starting the GPU container."* |
| `processing` | *"Modal T4 GPU is running our pipeline — audio detection, YOLO vision, ffmpeg splicing."* |
| `complete` | *"Done. Highlights extracted, uploaded to 0G Storage, captioned by 0G Compute."* |

**While waiting:** open the Modal dashboard and show the live function call in the Logs tab — real GPU execution visible in real time.

### 4. Show the results (~1 min)
- Each clip has a `root_hash` / CID on 0G Storage — show the 0G explorer link.
- Each clip has an AI-generated caption from 0G Compute Router.
- **Say:** *"Every highlight is permanently stored on 0G's decentralized network. The captions are generated by minimax-m3 running inside 0G Compute's TEE-attested environment — verifiable AI inference."*

### 5. The pitch close (~30s)
*"ZeroFrame runs entirely on decentralized and serverless infrastructure. No centralized video platform, no opaque AI black box. The entire pipeline — from upload to highlight detection to captioned clips — is verifiable, permanent, and runs for free at this scale."*

## Pre-demo checklist

Run these **5 minutes before** presenting:

```powershell
# 1. Confirm backend is awake (not sleeping)
curl https://zeroframe-backend.onrender.com/health

# 2. Confirm Modal worker is deployed
# Check: https://modal.com/apps/aaryan-sharma-5/main/deployed/zeroframe-worker

# 3. Open frontend in browser, verify it loads

# 4. Have a short test clip ready to upload (30-60s = faster demo)
```

> [!TIP]
> Render free tier sleeps after 15 min of inactivity, but the keep-alive ping prevents this.
> If you haven't hit the backend recently, do a manual `curl /health` 60 seconds before the
> demo to guarantee it's warm.

> [!TIP]
> Modal has a cold start of ~10–15s on first invocation (container spin-up). If the demo is
> time-pressured, trigger a dummy run via `modal run worker/modal_worker.py::run_pipeline`
> beforehand to pre-warm it.

## Troubleshooting

| Problem | Fix |
|---|---|
| `/health` returns `modal_configured: false` | `MODAL_ENDPOINT` not set in Render → check Environment tab |
| `/health` returns `redis_ok: false` | Render Redis sleeping → hit it once, retry |
| Job stuck at `pending` > 2 min | Modal cold start — wait, or check Modal logs at modal.com |
| Job stuck at `processing` | Check Modal function logs for Python errors |
| Upload fails (502) | 0G Storage testnet may be down — check `indexer-storage-testnet-turbo.0g.ai` |
| Vercel shows old API URL | Redeploy Vercel after setting `NEXT_PUBLIC_API_URL` |
